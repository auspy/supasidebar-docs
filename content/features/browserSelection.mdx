# Browser Selection

Choose which browser opens your saved links - assign specific browsers per link, use your active browser dynamically, or pick on demand for ultimate flexibility.

## When to Use

- **Context-based browsing**: Open work links in Chrome, personal in Safari, dev in Firefox
- **Dynamic workflows**: Use whichever browser you're currently in without manual switching
- **One-time choices**: Pick a browser for a specific session without changing defaults
- **Browser-specific features**: Use browsers that work best for certain sites (Arc for design, Safari for Apple sites)
- **Multi-account management**: Separate work and personal accounts across browsers
- **Testing workflows**: Open same link in different browsers for cross-browser testing

## Browser Opening Behaviors

### Assigned Browser (Default)

Each link remembers which browser it should open in.

**How It Works:**
1. Save a link from Chrome ‚Üí Automatically assigned to Chrome
2. Can change assignment: Right-click ‚Üí Set Browser ‚Üí Choose new browser
3. Opening that link always uses assigned browser

**Best For:**
- Consistent workflows
- Site-specific browser requirements
- Organizing by context (work vs. personal)

[PLACEHOLDER: Screenshot showing link with browser badge indicating Chrome]

**Example Workflow:**
- Gmail ‚Üí Always Chrome (work account)
- iCloud ‚Üí Always Safari (personal account)
- GitHub ‚Üí Always Firefox (development)
- Each link opens in its designated browser

### Active Browser Mode

Links open in whichever browser is currently active (frontmost).

**Enable:**
1. Open **Preferences ‚Üí General**
2. Set **Link Opening Behavior** to **Use active browser**

**How It Works:**
- Click link while Chrome is active ‚Üí Opens in Chrome
- Switch to Safari, click same link ‚Üí Opens in Safari
- Dynamic behavior based on current browser

**Best For:**
- Research workflows with single browser
- Avoiding browser switching
- Testing links in multiple browsers
- Ad-hoc browsing sessions

[PLACEHOLDER: Diagram showing same link opening in different browsers based on active browser]

**Example Workflow:**
1. Working in Chrome all day
2. All links open in Chrome automatically
3. Switch to Firefox for testing
4. Same links now open in Firefox
5. No manual browser selection needed

### Always Ask Mode

Browser picker appears every time you open a link.

**Enable:**
1. Open **Preferences ‚Üí General**
2. Set **Link Opening Behavior** to **Always ask**

**How It Works:**
- Click any link
- Browser picker popover appears
- Choose browser
- Link opens

**Best For:**
- Users who like explicit control
- Frequently switching between browsers
- Testing and development workflows
- Indecisive browser preferences

[PLACEHOLDER: Screenshot showing browser picker popover with all 14 browsers]

**Example Workflow:**
1. Click link in sidebar
2. See browser picker: Safari, Chrome, Firefox, Arc...
3. Choose Firefox for this time
4. Link opens in Firefox
5. Next link, repeat choice

## Browser Window Adjustment

Automatically resize browser windows to prevent sidebar overlap.

### Enable Window Adjustment

1. Open **Preferences ‚Üí General**
2. Set **Browser window adjustment** to desired mode:
   - **None**: No adjustment
   - **Minimal**: Small adjustment to prevent overlap
   - **Fill**: Maximize browser to fill remaining space

[PLACEHOLDER: Before/after screenshots showing browser window adjustment with sidebar visible]

### Adjustment Modes

**None:**
- Browser windows not adjusted
- Sidebar may overlap browser content
- Use if you prefer manual window management

**Minimal:**
- Browser window shrinks slightly
- Just enough to prevent sidebar overlap
- Preserves most of your browser width
- Recommended for most users

**Fill:**
- Browser window maximized to fill available space
- Sidebar and browser perfectly tiled
- Optimal screen space usage
- Great for split-screen workflows

[PLACEHOLDER: Diagram showing three adjustment modes side-by-side]

### Supported Browsers

Window adjustment works with:
- Chrome
- Edge
- Brave
- Vivaldi
- Safari (limited)
- Other Chromium-based browsers

**Not Supported:**
- Firefox (window management API limitations)
- Arc (custom window behavior)
- Zen (Firefox-based)

## Troubleshooting

### Browser Not in Picker

**Issue:** Expected browser doesn't appear in picker

**Solutions:**
1. Ensure browser is installed in Applications folder
2. Verify browser is supported (see [Browser Support](browserSupport))
3. Restart SupaSidebar
4. Check browser app name matches expected name

### Wrong Browser Opens

**Issue:** Link opens in unexpected browser

**Solutions:**
1. Check link's browser assignment (right-click ‚Üí check which browser is selected)
2. Verify Link Opening Behavior in Preferences ‚Üí General
3. If using "Active Browser" mode, ensure correct browser is frontmost
4. Right-click link ‚Üí Set Browser ‚Üí Choose correct browser

### Browser Picker Doesn't Appear

**Issue:** ‚å•Click or ‚å•Return doesn't show browser picker

**Solutions:**
1. Ensure Option key is held down while clicking/pressing Return
2. Check keyboard is working (test Option key in other apps)
3. Verify link is selected (for keyboard method)
4. Try mouse method if keyboard doesn't work
5. Restart SupaSidebar

### Window Adjustment Not Working

**Issue:** Browser window doesn't resize with sidebar

**Solutions:**
1. Check Preferences ‚Üí General ‚Üí Browser window adjustment is not "None"
2. Verify browser supports window adjustment (see supported browsers above)
3. Firefox and Arc don't support automatic adjustment
4. Try manual window resizing if automatic doesn't work

### Auto-Visibility Not Triggering

**Issue:** Sidebar doesn't appear when switching to browser

**Solutions:**
1. Check Preferences ‚Üí General ‚Üí "Show sidebar when browser is active" is enabled
2. Ensure browser has Automation permissions
3. Try clicking browser window to make it frontmost
4. Restart SupaSidebar

### Browser Assignment Not Saving

**Issue:** Browser selection doesn't persist

**Solutions:**
1. Ensure you're using "Set Browser" (right-click), not one-time selection (‚å•Click)
2. Check you're not in "Always ask" mode (Preferences ‚Üí General)
3. Verify link saves properly (delete and re-save if needed)
4. Check CloudKit sync status if using multiple devices
